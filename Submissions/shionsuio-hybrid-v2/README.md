# Hybrid SDRF Extraction Pipeline

Rule-based and LLM-assisted pipeline for the Kaggle "Harmonizing the data of your data" competition.

## Overview

This submission combines three sources of information:

1. PRIDE project metadata as high-confidence anchors.
2. LLM extraction from publication text.
3. Rule-based per-file assignment from raw filenames plus manual overrides.

The final `submission.csv` is generated by `pipeline/build_submission_v2.py`.

## Method

### 1. PRIDE Anchors

`fetch_pride_anchors.py` queries PRIDE and stores:

- organism
- instrument
- disease
- organism part
- modifications
- raw file names

These anchors are used as high-priority values for stable fields such as
`Characteristics[Organism]` and `Comment[Instrument]`.

### 2. LLM Extraction

`extract_with_llm_v2.py` runs a staged extraction process over paper sections:

- overview
- sample preparation
- MS settings
- raw-file interpretation

The extraction outputs are stored as JSON files under `pipeline/llm_extractions/`.
This package ships with precomputed outputs so the final submission can be
rebuilt without re-querying the API.

### 3. Submission Assembly

`build_submission_v2.py` merges:

- LLM outputs
- PRIDE anchors
- training-derived vocab and non-NA priors
- rule-based supplements from text and filenames
- final hand overrides

This script writes `submission.csv`.

## Usage

### Rebuild From Bundled Intermediate Files

```bash
pip install -r requirements.txt
python pipeline.py --rebuild-only
```

### Optional: Refresh PRIDE Anchors

```bash
python pipeline.py --run-anchor-fetch
```

### Optional: Refresh LLM Extractions

Requires `ANTHROPIC_API_KEY`.

```bash
export ANTHROPIC_API_KEY=...
python pipeline.py --run-llm-extract
```
